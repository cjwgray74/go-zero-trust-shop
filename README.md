# Go Zero‑Trust Shop — Vault AppRole + Dynamic PostgreSQL + Terraform + Podman Machine (Windows)

This project demonstrates a production‑style Zero‑Trust microservice architecture using:

- Vault (AppRole authentication and dynamic PostgreSQL credentials)
- Terraform for managing all Vault configuration (AppRole, DB engine, roles, policy)
- Podman Machine for containerized Vault and PostgreSQL on Windows
- Go service (`auth-svc`) with embedded SQL migrations, pgx, and CRUD endpoints
- One‑click development script (`start-dev.ps1`) that brings the entire stack up automatically

The system requires **no static secrets**, **no stored passwords**, and the service receives a
**new short‑lived database identity each time it runs**.

---

## Key Features Implemented

### 1. Vault AppRole Authentication
The service authenticates to Vault using:
- `VAULT_ROLE_ID`
- `VAULT_SECRET_ID`

These are **generated by Terraform**, not stored in code or source control.

### 2. Dynamic PostgreSQL Credentials
Vault’s Database Secrets Engine issues:
- short‑lived usernames/passwords
- with TTL control
- automatically rotated
- with least‑privilege grants (`CONNECT`, `USAGE`, `CREATE`, CRUD on all tables)
